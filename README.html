🗣️ debate-board 複数のAIと人間が同じボードで議論して、結論をカンバンに残すローカルアプリ

依存パッケージ 0個（npm install 不要） APIキー不要 議論データは PC の外に出ない 127.0.0.1 専用

お題を入力→ ONの参加者が順番に発言→ 全員がカードを操作→ 📋 結論サマリ生成
✅ decided決定事項
💬 discussing議論中の論点
⏸ held保留

必要なもの

必須Node.js 20以上（それだけ。npm installは存在しない）
参加AI
（使う分だけ）		 Claude Code CLI（claude）／ Codex CLI（codex）／ Grok Build CLI(grok)
各CLIのサブスク認証をそのまま流用（追加API課金なし）。
Ollama や LM Studio 等のOpenAI互換サーバがあればローカルLLMも参加可

CLIが1つも無くても、Ollama＋あなた（human）だけで議論できます（AI相当の参加者が最低1人いればOK。開始にはON参加者2人以上が必要）。

🚀 クイックスタート（GUI）

git clone https://github.com/yushiesah7/debate-board.git
cd debate-board
copy config.example.json config.json   # macOS/Linux は cp
node src/server.mjs

→ ブラウザで http://127.0.0.1:8787 を開いて、お題を入れて「開始」。

画面でできること

  • 参加者チップをクリックで ON/OFF（ONの人だけにターンが回る）
  • カードをドラッグでレーン移動、クリックで編集、「＋」で追加
  • 右ペインで発言ログと各参加者のNOTEを閲覧（自分のNOTEは編集可）
  • 「あなた」がONのとき、自分のターンで発言 or スキップ
  • 終了すると📋 結論サマリがボード上部に表示

※ サーバ無しで public/index.html を直接開くとモックモード（サンプルデータでUIだけ試せる）。

⏹ 停止方法

  • サーバを起動したターミナルで Ctrl+C（これだけ）
  • ターミナルを閉じてしまった場合（PowerShell）:
    Get-NetTCPConnection -LocalPort 8787 -State Listen | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
  • ⚠ 起動はこのリポジトリのフォルダに cd してから。Cannot find module ...\src\server.mjs が出たら起動場所が違います

💻 ターミナル派（CLIモード）

node src/cli.mjs "きのこの山 vs たけのこの里、どちらが至高か" 3

GUI無しでAIだけの議論を実行し、各ターンの発言・カード操作・最後の結論サマリをそのまま表示します（human参加者は自動でOFF）。

⚙️ 参加者のカスタマイズ（config.json）

{
  "port": 8787,
  "maxRounds": 4,
  "participants": [
    { "id": "nagi",  "name": "凪",        "adapter": "claude", "model": "sonnet", "pcAccess": "read", "enabled": true },
    { "id": "aki",   "name": "アキ",      "adapter": "codex",  "pcAccess": "read", "enabled": true },
    { "id": "roki",  "name": "ロキ",      "adapter": "grok",   "pcAccess": "read", "enabled": true },
    { "id": "local", "name": "ローカルLLM", "adapter": "ollama", "model": "qwen3",
      "endpoint": "http://127.0.0.1:11434", "enabled": false },
    { "id": "you",   "name": "あなた",    "adapter": "human",  "enabled": false }
  ]
}
adapter実体備考
claudeClaude Code CLImodel でモデル指定可
codexCodex CLIChatGPTログインを流用
grokGrok Build CLIコーディング用の重いsystem promptは自動回避
ollamaOllama HTTP APIendpoint 必須。ローカルLLM
openai-compatOpenAI互換 /v1/chat/completionsLM Studio / llama.cpp server / vLLM 等
humanGUIの入力欄1人まで。ターンが来ると入力待ちUIに

人数・組み合わせは自由（AI 5人の乱闘も可）。ターン順は配列の並び順です。

🔑 pcAccess — AIにPCをどこまで見せるか（CLI系のみ）

"read"（既定）AIがPC内のファイルを読んで議論の根拠にできる（例:「このリポの設計を踏まえて意見して」）。書き込み・実行は不可
"full"読み書き・コマンド実行まで許可。明示オプトイン・自己責任（信頼できるお題でだけ推奨）

ollama / openai-compat / human はHTTP・GUIのみなので対象外。プライバシー重視ならCLI自体の権限設定でも縛れます。

🔒 プライバシー

  • お題・発言・カード・議事録はすべて state/ に保存 — git管理外で、リポジトリに載ることはありません（CIでも二重ガード）
  • サーバは 127.0.0.1 のみにバインド。外部からアクセス不可
  • コードにAPIキー・秘密情報は一切含まれません（各CLIのローカル認証を利用）

🩹 困ったとき

症状対処
開始ボタンが押せないONの参加者が2人未満。チップをONに
ある参加者がずっと (pass)そのCLIが未インストール/未ログイン。claude --version 等で確認。エラー内容は発言ログに表示
ollamaが応答しないOllama起動と endpoint・model 名を確認
ポート衝突config.json の port を変更
